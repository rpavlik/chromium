Known Bugs

Not everything works as advertised in Chromium, unfortunately.  Following is a list of issues I'm aware of at the time of this release:

  • SPU's cannot change their exported API on the fly.
  • Display lists don't work with the Tilesort SPU.  This is related to the previous bug.
  • Flow control has some of the same issues that it did in WireGL.  It's fairly easy for a small parallel submission program to overflow the memory of a server, particularly over TCP/IP.  The "psubmit" program shows off this memory explosion nicely.  This is partially due to the fact that the client doesn't exactly know how much memory a message will consume on the server, and partially due to the new networking abstraction (I think).  This is a very high priority bug.
  • DrawPixels/ReadPixels are not implemented.
  • Evaluators aren't implemented in the Tilesort SPU.
  • OpenGL Selection/Feedback buffers don't work.
  • The Print SPU doesn't special-case many functions, so often a pointer address is printed when you're interested in the contents of the data.  I've been updating the Print SPU as my debugging needs have dictated.
  • Byte swapping isn't implemented, so a cluster has to be homogenous with respect to endianness.
  • The networking abstraction still isn't as robust as I need to create network connections outside of the client/server model.
  • The Render SPU will put up a new window even when it's bolted directly to an application, which is a little awkward.  This is a high priority bug.
  • glPushAttrib and glPopAttrib are not implemented, although the framework to implement those calls exists.
  • pixel data specified with GL_LSB_FIRST or GL_SWAP_BYTES cannot be unpacked.
  • Certain glGet queried parameters do not reflect the actual state of the graphics card on the other end of a network (for example, the maximum texture size)
  • The mothership does not start jobs for you; you have to start them manually
  • Adding multiple tiles to a server is (very) partially implemented, and that part is untested.  Consequently Lightning-2 is not supported.
